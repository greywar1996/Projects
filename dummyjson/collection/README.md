# Postman Collection — API Testing (E-commerce / DummyJSON)

**Описание:**

Данная коллекция Postman предназначена для тестирования REST API интернет-магазина (на основе DummyJSON).

**Цель коллекции — продемонстрировать:**
- понимание структуры API
- базовые проверки ответа сервера
- работу с параметрами запроса
- проверку бизнес-логики
- негативные сценарии
- кросс-валидацию данных с SQL

Коллекция используется в связке с SQL-запросами для проверки согласованности данных между API и базой данных.

**Используемые инструменты:**
- Postman
- JavaScript (Tests)
- PostgreSQL
- SQL (JOIN, LEFT JOIN, GROUP BY, DISTINCT, SUM, COUNT)
- VS Code (анализ JSON)

**Коллекция включает следующие группы запросов:**
1. Products:
GET /products
- Проверка структуры ответа
- Проверка типов данных (id, title, price, category)
- Проверка непустого массива

2. Фильтрация и уникальные значения:
- Анализ категорий
- Сравнение уникальных значений с SQL DISTINCT
- Проверка соответствия количества записей

3. Carts:
GET /carts
- Проверка агрегированных данных (total)
- Сравнение с SQL SUM

4. Users:
- Проверка связей между пользователями и корзинами
- Валидация через JOIN

5. Негативные сценарии:
- Запрос несуществующего ресурса
- Проверка корректного HTTP-кода
- Проверка структуры ошибки

**Примеры реализованных проверок:**
- Проверка статус-кода (200, 404)
- Проверка структуры JSON
- Проверка типов данных
- Проверка непустого массива
- Извлечение полей из ответа
- Анализ уникальных значений
- Сравнение данных с SQL

**Подход к тестированию:**
- Выполнение запроса через Postman
- Анализ структуры JSON
- Выделение необходимых полей (например, category)
- Сравнение результатов с SQL-запросами
- Документирование расхождений

В ряде случаев использовались базовые Postman test scripts для автоматизации повторяющихся проверок.

**Что демонстрирует данная коллекция:**
- Понимание REST API
- Навык анализа JSON-структур
- Умение проверять бизнес-логику
- Понимание реляционных связей
- Навык кросс-валидации API и БД
- Аккуратное документирование

**Как запустить:**
- Импортировать JSON-коллекцию в Postman
- Установить переменную baseUrl
- Выполнить запросы вручную или через Runner
- Проверить вкладку Tests и Console
