# Баг-репорты — API тестирование (DummyJSON)

**Описание:**

В данном разделе представлены баг-репорты, составленные в процессе тестирования REST API (DummyJSON).

**Цель раздела — продемонстрировать:**
- умение находить отклонения в работе API
- корректно оформлять дефекты
- описывать воспроизводимые шаги
- анализировать ожидаемый и фактический результат

**Тестирование проводилось для следующих API-эндпоинтов:**
- GET /products
- GET /products/{id}
- GET /products/category/{category}
- GET /carts
- GET /carts/{id}
- GET /users/{id}

**Проверялось:**
- Корректность HTTP-статусов
- Структура JSON-ответа
- Типы данных
- Логика фильтрации
- Корректность сортировки
- Обработка несуществующих ресурсов
- Соответствие данных между API и БД

**Структура баг-репорта:**
- ID
- Название
- Эндпоинт
- Метод запроса
- Предусловия
- Шаги воспроизведения
- Ожидаемый результат
- Фактический результат
- Severity
- Priority

**В рамках тестирования были описаны:**
1. Логические дефекты
- Некорректная фильтрация
- Несоответствие количества записей
- Ошибка сортировки

2. Структурные дефекты
- Отсутствие обязательных полей
- Некорректные типы данных
- Неполная структура ответа

3. Негативные сценарии
- Запрос несуществующего ID
- Некорректный HTTP-код
- Отсутствие корректной обработки ошибки

P.S. DummyJSON является mock API.
Часть багов зафиксирована в учебных целях и демонстрирует подход к анализу и документированию дефектов.
Цель — показать методологию тестирования, а не выявление реальных production-ошибок.

**Что демонстрирует данный раздел:**
- Понимание принципов REST API
- Умение анализировать структуру ответа
- Навык оформления дефектов
- Логическое мышление
- Внимательность к деталям
