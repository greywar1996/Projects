# Тест-кейсы — API тестирование (DummyJSON)

**Описание:**

В данном разделе представлены тест-кейсы, разработанные для тестирования REST API сервиса DummyJSON.

**Цель — продемонстрировать:**
- умение проектировать тест-кейсы для API
- понимание структуры JSON-ответов
- проверку бизнес-логики
- проверку корректности фильтрации и обработки ошибок
- системный подход к тестированию backend

**Тестирование проводилось для следующих эндпоинтов:**
- GET /products
- GET /products/{id}
- GET /products/category/{category}
- GET /carts
- GET /carts/{id}
- GET /users/{id}

**Область покрытия:**
1. Проверка структуры ответа
- Наличие обязательных полей
- Проверка типов данных
- Проверка непустого массива

2. Проверка фильтрации
- Корректность фильтрации по категории
- Проверка соответствия параметра запроса и данных в ответе

3. Проверка получения ресурса по ID
- Успешное получение существующего ресурса
- Обработка запроса несуществующего ID

4. Проверка агрегированных данных
- Проверка total в carts
- Сравнение с расчётами в БД (SQL)

5. Негативные сценарии
- Запрос несуществующего ресурса
- Проверка HTTP-кода (404)
- Проверка структуры ошибки

**Структура тест-кейса:**
- ID
- Tite
- Эндпоинт
- Метод
- Предусловия
- Шаги
- Ожидаемый результат

**При разработке тест-кейсов использовались следующие принципы:**
- Проверка позитивных сценариев
- Проверка негативных сценариев
- Проверка граничных значений
- Проверка бизнес-логики
- Проверка целостности данных

Часть проверок дополнительно валидировалась через SQL-запросы к локальной базе данных.

**Особенности проекта:**
- Тест-кейсы разработаны вручную
- Покрывают ключевые API-эндпоинты
- Сфокусированы на backend-логике
- Дополнены кросс-проверкой через SQL

P.S. DummyJSON является mock API.
Тест-кейсы разработаны в учебных целях для демонстрации подхода к API-тестированию.
