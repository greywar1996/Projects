# SCHOOL-1 — Тестирование REST API (CRUD + Авторизация)

**Описание:**

Учебный проект по тестированию REST API системы управления задачами (Todos).

**Цель проекта — отработать:**

- авторизацию
- работу с переменными окружения
- CRUD операции
- позитивные и негативные сценарии
- граничные значения
- проверку структуры ответа API

**Используемые инструменты:**

- Postman
- REST API
- Bearer Token авторизация
- JavaScript (Postman test scripts)

**Реализовано:**

### 1. Авторизация

- Регистрация пользователя
- Логин
- Сохранение token и user_id в environment
- Использование Bearer Token на уровне коллекции

### 2. Позитивные сценарии

- Получение списка задач
- Создание задачи
- Редактирование задачи
- Удаление задачи

Проверяется:
- статус-коды
- структура ответа
- наличие обязательных полей
- корректность переданных данных

### 3. Негативные сценарии

Проверены:

- длина title = 0
- длина title > допустимого
- отсутствие обязательных полей
- неверный тип данных
- редактирование чужой задачи
- удаление несуществующей задачи

Проверяются корректные HTTP-коды ошибок (400, 403, 404).

**Как запустить:**

1. Импортировать коллекцию в Postman
2. Создать Environment
3. Указать base_url
4. Выполнить регистрацию
5. Выполнить авторизацию
6. Запускать тестовые сценарии

**Результат:**

Проект демонстрирует:

- понимание REST архитектуры
- работу с авторизацией
- умение строить позитивные и негативные тест-кейсы
- проверку структуры ответа API
- работу с переменными и зависимыми запросами
